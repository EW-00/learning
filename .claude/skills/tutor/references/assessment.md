# 循环 C · 评估：出题、判卷、过门

## 目标写法（roadmap 和考核共用）

- 只用可观察行为动词：推导、建模、诊断、比较、重构、实现、解释给新手听
- 禁用：了解、理解、掌握、熟悉（不可测）
- 句式：「给定 <条件>，能 <行为>，达到 <标准>」

## 出题五规则

1. 每题绑定 ROADMAP 里的一条行为目标
2. 过门题用**同构平行题**：深层结构相同、表面情境必换（换数值、换背景故事、换领域皮）
3. 每模块至少 1 道 near transfer（同结构新情境）；每 2–3 模块至少 1 道 far transfer（换表征或跨领域应用）
4. 诊断题沿依赖图选，最大化信息量（答对/答错都能剪枝）
5. 开放任务必配 rubric：准则 ≤7 条、3–5 档、每档描述可观察行为

## 课内软门（小节级）

- 小节末 1–3 题快速验证；小缺口**不拦路**：记入 `ERRORS.md`（格式见下），继续推进
- ERRORS.md 条目在 2 天后、1 周后由复习引擎自动复验（平行题）；连续两次复验通过才销账

ERRORS.md 条目格式：

```markdown
## E-014 | 2026-07-06 | 模块: branch-and-bound
- 误概念：认为 LP 松弛给出的是原问题的下界（最大化问题中应为上界）
- 证据：练习 3.2 中定界方向写反
- 矫正：用「放松约束只会让最优值变好或不变」的几何直觉重讲
- 复验：[ ] 2026-07-08  [ ] 2026-07-13
```

## 模块硬门（过门考试）

**阈值**：`速成应用` 85% / `系统深学` 90% / `兴趣考级` 按考纲及格线但建议 90%。

流程：

1. **考前密封出卷**（护栏 4）：先自己完整解出每道题，将「题目 + 标准解答 + 逐条 rubric」写入 `exams/<module>-A.md`，然后才向她出题。同时生成 B 卷（平行题）备重测。
2. **考试**：闭卷、无提示、hint ladder 关闭。明确告知「这是过门考，我不会给提示」。含平行题 + 迁移题（见出题规则）。
3. **判卷（护栏 3，反 sycophancy 协议）**：
   - 派一个**独立子 agent** 评分，输入只有三样：作答原文、rubric、预置解答——不含对话历史，不含她的任何自我评价
   - 逐条 rubric 打分并给出依据；预期 LLM 偏松，边界情况从严
   - 主会话把子 agent 的评分结果如实转达，不加缓冲修饰
4. **不达标** → 矫正循环：只针对错的知识组件，换表征重教（护栏 7）→ 用 B 卷中对应的平行题重测（护栏 8）→ 通过后明确宣告「此模块已掌握」。B 卷也没过 → 回到教学模式，怀疑先修有洞，向依赖图上游排查。
5. **已达标** → 不空等：给深化任务（far transfer / 小项目）或直接进入下一模块。

## 无辅助检查点（护栏 9）

每 2–3 个模块，布置一次完全独立的作业：题目发给她，约定她**不用任何 AI 辅助**完成后带回来评。这是依赖度监测的唯一可靠信号：独立表现明显低于会话内表现 → 说明前面给的辅助太多，降速、加大独立练习比例、收紧 hint ladder。

## 期望管理

向她汇报进度时用「不留漏洞的稳定进步」的语言，不用「超越 98% 的人」的神话。真实的量级：坚持使用的现实收益约 0.3–0.8σ（VanLehn 2011; Nickow 2020）。汇报要诚实：不稳的模块直说不稳。
