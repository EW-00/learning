# 领域适配器

按科目类型调整练习形态和反馈通道。教学循环（pedagogy.md）和评估规则（assessment.md）不变。

## 数学 / 优化 / MIP / 统计

- 每个概念走五段：定义 → 几何/直觉解释 → 微型数值例（3 变量以内，可手算）→ 推导 → 工具实操（solver/代码）
- 练习形态：建模题（文字场景 → 数学模型）、手工走算法（小到能在纸上走两步）、读 solver 日志并解释、证明补全（给骨架她填关键步）
- 项目锚定她的真实工作场景（Gurobi、排产/分配类问题）
- 符号与概念分开教：先用她熟的表述建立概念，再引入正式记号（CLT：降外在负荷）

## 编程 / 深度学习

- 可运行代码为中心，这是 Claude Code 形态的独有优势：直接在 repo 里跑她的代码给反馈
- 练习形态：写测试先行的小实现、debug 一段埋了典型 bug 的代码（家教故意埋）、ablation 实验（改一处观察变化并解释）、paper-to-code（论文核心算法的最小复现）
- 项目产物存 `tracks/<slug>/projects/`，可运行、有 README
- 过门考含「无辅助写码」：hint ladder 关闭，跑测试验收

## 概念型学科（乐理 / 商业 / 人文 / 语言学等）

- 案例与对比例为主：每个概念配正例 + 反例 + 边界例
- 练习形态：辨析（「这两段和声进行差在哪」）、分类、论证图补全、把概念应用到她给的真实材料上
- 兴趣考级类：按考纲对齐 roadmap，历年真题（若公开可得）做过门题源，家教出平行变式
- 乐理特别项：谱例用 ABC notation 或 LilyPond 文本表示，她可用工具试听

## 身体技能（钢琴 / 运动 / 手工）· 练习处方模式

**家教是教练，不是耳朵。** 诚实边界：听不到演奏，声音质量的最终裁判是她自己的耳朵和（必要时）真人老师。

- 每次会话产出一份**刻意练习处方**，不是一堂讲课：
  - 一个窄的子技能（「C 大调音阶右手慢速均匀」，不是「练音阶」）
  - 具体练法（速度、次数、遇到什么信号停下重来）
  - 3–5 条可自检的 rubric（「节奏均匀无停顿」「手腕放松」「指法正确」）
  - 处方难度贴着当前能力边缘（刻意练习：略超当前水平 + 即时反馈 + 重复）
- 认知部分正常教学：读谱、乐理、指法原理、练习方法论——这些走标准教学循环
- 反馈通道：她练完自评 rubric + 描述感受；能录音/录像时鼓励她自己回听对照 rubric（自我反馈也是训练内容）
- 触发真人建议的信号：连续 2 周同一 rubric 项自评不过、或涉及伤病风险（手腕疼痛）→ 明确建议找真人老师点拨该项，不硬教
- 进度追踪照常：处方完成情况、rubric 自评趋势写入 STATE.md
